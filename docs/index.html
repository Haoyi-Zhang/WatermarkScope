Research gap

The gap is not only detection. It is trustworthy evidence after code leaves the prompt.

Generated code may be copied, edited, translated, or deployed outside the original session. At that point, a detector score alone is too weak unless the counted rows, controls, and claim boundary are clear.

01

Code moves

The generated snippet may no longer have its original prompt, model session, or owner context.

02

Code must run

For source code, a watermark claim is weak if the benchmark row is not executable and countable.

03

Claims need limits

White-box recovery, black-box audit, owner attribution, and security triage should not share one score.

Why this matters Executable rows protect the denominator.

This fixes exactly which code-generation rows were counted before interpreting any watermark signal.

Design decision

The project treats watermarking as an evidence problem: a result is interpreted only after the counted rows, controls, artifact, access model, and claim boundary are fixed.

Method

The method is an evidence contract plus a five-stage pipeline.

The method is simple to explain: admit an evidence object first, then interpret the watermark result only inside that admitted scope.

What is counted?

A result only means something after the denominator is fixed.

For example, the submitted SemCodebook surface reports 23,342 recoveries over 24,000 admitted positive rows. Misses remain inside the denominator; they are not hidden by the aggregate.

Limits and examiner-safe claims

The strongest part of the work is that it says what it does not prove.

No universal watermark claim

SemCodebook supports admitted white-box cells; it is not a universal natural-generation theorem.

No provider accusation

CodeDye reports conservative black-box audit evidence; it is not contamination prevalence or wrongdoing evidence.

No general authorship proof

ProbeTrace is scoped active-owner verification under a fixed registry and split.

No safety certificate

SealAudit is selective triage with abstention; it is not automatic harmlessness certification.
