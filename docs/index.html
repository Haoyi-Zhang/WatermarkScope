WatermarkScope

A benchmark-to-audit framework for source-code watermarking in code generation models.

Instead of reporting one detector score, WatermarkScope records what evidence was admitted, what controls were run, what artifact can be inspected, and what claim boundary follows.

Problem framing

Generated code leaves the chat window. Watermark evidence has to survive that movement.

01

Code is copied, edited, translated, and reused.

A watermark claim cannot assume the original generation session is still intact.

02

A score is not yet evidence.

The score must be tied to a task population, controls, access model, and preserved artifact.

03

Abstention protects the claim.

When a broader claim is not supported, WatermarkScope records a bounded null or review outcome.

Interactive pipeline

Each stage answers one question and stops at its admitted boundary.
