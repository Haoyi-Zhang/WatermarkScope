**English translation:**  
Good afternoon. I am Haoyi Zhang. My final year project is WatermarkScope: A Benchmark-to-Audit Framework for Source-Code Watermarking in Code Generation Models. Today I will not use a traditional PowerPoint. I will use this project homepage as my presentation route. I will move page by page, and finally stop on the Q&A page. The core problem of this project is simple. Code generated by large language models can be copied, edited, translated, and used in other projects. In that situation, if we only report one watermark detector score, the score is hard to interpret and hard to audit. So my work is not only about getting a higher detection score. It is about making source-code watermarking results inspectable, reproducible, and bounded.

### 2. Problem

**中文演讲稿：**  
我选择这个题目的原因是，代码水印和普通文本水印不完全一样。代码不只是文字，它还要能运行，要有语义，要能经过格式化、重命名、翻译或者局部修改之后仍然被解释。比如，一个代码片段可能从原始 prompt 会话中离开，然后进入 GitHub、课程作业、企业项目或者其他模型生成的上下文里。这个时候我们不能只问“检测器有没有命中”，还要问：这个结果到底数了哪些样本？有没有负控？有没有正控？证据文件在哪里？这个结果能支持什么结论，又不能支持什么结论？

**English translation:**  
The reason I chose this topic is that code watermarking is not the same as normal text watermarking. Source code is not just text. It needs to run, it has semantics, and it may go through formatting, renaming, translation, or local editing. For example, a generated code snippet may leave the original prompt session and later appear in GitHub, coursework, an industrial project, or another model-generated context. At that point, we should not only ask whether a detector fires. We should also ask: what samples were counted, what negative controls were used, what positive controls were used, where the evidence files are, what conclusion the result can support, and what conclusion it cannot support.

**中文演讲稿：**  
所以我的研究空白不是简单的“有没有检测方法”。更关键的空白是：当水印结果要被老师、审稿人或者工程人员审查时，我们怎样把它组织成可信的证据。不同场景的证据边界也不同。白盒溯源、黑盒审计、所有者归因和安全分诊，它们回答的是不同问题。如果把这些结果全部合成一个总准确率，反而会掩盖真正重要的信息。

**English translation:**  
So the research gap is not simply whether a detection method exists. The more important gap is how to organize watermarking results as credible evidence when they need to be checked by supervisors, reviewers, or engineers. Different settings also have different claim boundaries. White-box provenance recovery, black-box audit, owner attribution, and security triage answer different questions. If we merge all of them into one overall accuracy score, we may hide the information that actually matters.

### 3. Method

**中文演讲稿：**  
我的方法可以概括为两个部分。第一部分是 evidence contract，也就是证据契约。第二部分是围绕这个证据契约实现的五个阶段。证据契约的作用是，在我解释任何一个实验结果之前，必须先回答五个问题：第一，分母是什么，也就是到底数了哪些样本；第二，控制组是什么；第三，证据文件在哪里；第四，这个结果假设了什么访问条件；第五，这个结果的 claim boundary 是什么，也就是它能说什么，不能说什么。

**English translation:**  
My method has two parts. The first part is an evidence contract. The second part is a five-stage implementation built around that contract. The purpose of the evidence contract is that before I interpret any experimental result, I must answer five questions. First, what is the denominator, meaning what samples are counted. Second, what controls are used. Third, where the evidence files are. Fourth, what access model is assumed. Fifth, what the claim boundary is, meaning what the result can claim and what it cannot claim.

**中文演讲稿：**  
在实现上，我把整个项目组织成五个阶段。CodeMarkBench 负责可执行 benchmark 的基础，它先保证评价对象不是随意挑出来的文本，而是可以被统计的代码任务。SemCodebook 是主要的白盒溯源方法，用结构化代码载体来恢复 provenance。CodeDye 是保守的黑盒审计，它不声称证明污染，而是保留受控条件下的稀疏信号。ProbeTrace 研究主动所有者归因，也就是在给定 owner registry 的情况下做有边界的归因。SealAudit 研究隐藏标记条件下的安全分诊，它允许 abstention，也就是证据不足时不强行判断。

**English translation:**  
In implementation, I organize the project into five stages. CodeMarkBench provides the executable benchmark foundation. It makes sure that the evaluation objects are not arbitrary text samples, but countable code tasks. SemCodebook is the main white-box provenance recovery method. It uses structured code carriers to recover provenance. CodeDye is a conservative black-box audit. It does not claim to prove contamination. It preserves sparse signals under controlled conditions. ProbeTrace studies active owner attribution, where the claim is bounded by a given owner registry. SealAudit studies security triage under hidden markers. It allows abstention, meaning it does not force a decision when the evidence is not enough.

### 4. Results

**中文演讲稿：**  
在结果页，我不会逐行读所有数字。我会用三个词来解释结果：denominator、control 和 boundary。首先，CodeMarkBench 完成了 140/140 个 canonical runs。这个结果说明 benchmark 表面是可执行、可统计的，但它不是在说所有水印都成功。它只是给后面的评价提供了一个固定分母。

**English translation:**  
On the results page, I will not read every number line by line. I will explain the results with three words: denominator, control, and boundary. First, CodeMarkBench completed 140 out of 140 canonical runs. This means the benchmark surface is executable and countable, but it does not mean every watermark succeeds. It provides a fixed denominator for later evaluation.

**中文演讲稿：**  
第二，SemCodebook 是我最核心的方法结果。在提交版毕业设计中，它在 24,000 个正例里恢复了 23,342 个，同时在 48,000 个负控里没有命中。这个结果能支持的结论是：在已经承认的白盒评估单元里，结构化 provenance recovery 是有效的。但我不会把它说成所有自然生成代码上的通用水印检测器，因为那会超过当前证据边界。

**English translation:**  
Second, SemCodebook is my main method result. In the submitted FYP version, it recovered 23,342 cases out of 24,000 positive cases, with zero hits in 48,000 negative controls. The conclusion this supports is that structured provenance recovery is effective inside the admitted white-box evaluation cells. But I do not describe it as a universal watermark detector for all naturally generated code, because that would go beyond the current evidence boundary.

**中文演讲稿：**  
第三，其他阶段展示了为什么边界很重要。CodeDye 在 300 个实时样本里只有 6 个稀疏信号，同时有 170/300 个正控和 0/300 个负控。这个结果不能被说成污染证明，只能说是保守的黑盒审计证据。ProbeTrace 完成了 300/300 个有边界的归因决策，并且在 1,200 个错误所有者控件里没有误报。SealAudit 在 960 个隐藏标记样本里有 81 个明确分诊结果，并且没有观察到 unsafe pass。整体结论是，可靠的代码水印研究不应该只看一个检测分数，而应该看分母、控制组、证据文件和边界。

**English translation:**  
Third, the other stages show why boundaries matter. CodeDye has only 6 sparse live signals out of 300 live samples, with 170 out of 300 positive controls and 0 out of 300 negative controls. This result should not be described as proof of contamination. It is conservative black-box audit evidence. ProbeTrace completed 300 out of 300 scoped attribution decisions, with zero false hits in 1,200 false-owner controls. SealAudit produced 81 decisive triage outcomes out of 960 hidden-marker samples, with no observed unsafe passes. The overall conclusion is that reliable source-code watermarking research should not depend on one detector score. It should report denominators, controls, evidence files, and claim boundaries.

## Q&A Answer Format

Use this rule in Q&A: answer directly first, then give one number or file, then state the boundary.
