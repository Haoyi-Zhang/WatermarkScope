# Canonical Release Inputs

This directory contains the single canonical execution inputs for the public release suite.

- `sources/` stores the exact normalized inputs executed by the benchmark workflow
- every reviewer-facing table, figure, and manifest in the public repo should align with these release inputs
- reviewer-facing execution truth is defined entirely by this release input layer

This release layer contains four public benchmark execution slices and three curated crafted benchmark families. The crafted families were finalized through manual release review, deterministic consistency checks, and public provenance records.

Execution-backed semantic-validation evidence is generated by the benchmark workflows and export pipeline; this pre-run release layer records finalized source truth and release provenance rather than embedded runtime result annotations.

The canonical dataset definitions, release sizes, language coverage tables, and review figures are documented in [`docs/datasets.md`](../../docs/datasets.md).
