Abstract (English) 摘要（英文）
Large language models increasingly generate source code that is copied, edited, translated, and deployed outside the original generation session. This project presents WatermarkScope, a benchmark-to-audit framework for evaluating source-code watermarking evidence under realistic code-generation conditions. The framework connects five stages: CodeMarkBench defines executable benchmark denominators; SemCodebook studies structured white-box provenance recovery; CodeDye preserves conservative black-box null-audit evidence; ProbeTrace evaluates scoped active-owner attribution; and SealAudit examines marker-hidden security triage. The current evidence surface includes 140/140 canonical CodeMarkBench runs, 23,342/24,000 SemCodebook recoveries with 0/48,000 negative-control hits, 6/300 CodeDye sparse live signals with 170/300 positive controls and 0/300 negative controls, 300/300 ProbeTrace scoped decisions with 0/1,200 false-owner controls, and 81/960 decisive SealAudit triage outcomes with 0 observed unsafe passes. The main conclusion is that source-code watermarking should not be judged by one detector score. Reliable evaluation requires fixed denominators, executable controls, uncertainty reporting, access-specific claim boundaries, and explicit abstention when evidence is insufficient.

Abstract (Chinese) 摘要（ 中文）
大型语言模型生成的源代码常被复制、修改、翻译并部署到原始生成会话之外，因此代码水印评估不能只依赖单一检测分数。本项目提出 WatermarkScope，一个面向代码生成模型水印证据的基准到审计框架。该框架包含五个阶段：CodeMarkBench 建立可执行基准与固定分母；SemCodebook 研究结构化白盒溯源水印；CodeDye 保留保守的黑盒空审计证据；ProbeTrace 评估有界的主动所有者归因；SealAudit 分析隐藏标记条件下的安全分诊。当前结果包括 CodeMarkBench 140/140 个规范运行，SemCodebook 23,342/24,000 个正例恢复且 48,000 个负控中 0 命中，CodeDye 在 300 个实时样本中得到 6 个稀疏信号并通过正负控验证，ProbeTrace 完成 300/300 个有界归因且 1,200 个错误所有者控件中 0 误报，SealAudit 在 960 个隐藏标记样本中给出 81 个明确分诊且未观察到不安全通过。结论是，可靠的代码水印研究需要固定分母、可执行控件、不确定性报告、访问模型边界以及证据不足时的显式弃权。
